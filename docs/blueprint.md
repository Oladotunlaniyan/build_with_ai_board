# **App Name**: GDGOC Unilorin Build With AI Demo Board

## Core Features:

- Homepage Project Display: Showcase approved AI projects in a responsive grid on the main board.
- User Authentication & Authorization: Enable users to register, log in/out, and assign distinct roles for regular users and administrators.
- Project Submission Workflow: A modal form allowing authenticated users to submit detailed project information, including screenshots, live URLs, and GitHub links.
- Admin Moderation Dashboard: Provide administrators with a dedicated interface to review, approve, or reject submitted student projects.
- Individual Project Details Page: A focused page to display all comprehensive information for a selected, approved project.
- AI Tool Tagging Assistant: Utilize an AI tool during project submission to suggest relevant 'AI tools used' or 'tech stack' based on the project's description.
- Theme Toggle: A user-facing control in the navigation bar to switch between the light and dark display modes of the application.

## Style Guidelines:

- The visual design draws inspiration from a blend of 'Academic and calm' with 'Developer-focused' aesthetics. A primarily light theme is established to prioritize readability, with colors carefully chosen for seamless adaptation to a dark mode.
- Primary color: A professional and serene deep indigo blue, facilitating clear readability and strong branding against light backgrounds (#334099).
- Background color: A subtle and calming off-white with a delicate hint of the primary hue, providing a clean and focused canvas (#ECEEF3).
- Accent color: A vibrant and engaging azure blue, designed to highlight interactive elements and provide visual interest (#289FDB).
- The sole font will be 'Inter' (sans-serif), selected for its modern, clean, and highly legible characteristics, perfectly suited for both headlines and extensive body text in an academic and developer-centric environment.
- Typography hierarchy is maintained by using 'Inter' in semibold for page titles, regular for body text, and smaller, muted styles for meta-information.
- The interface will feature generous padding, equivalent to 'p-6' or higher, ensuring ample white space and clear visual separation between all sections. Content will be horizontally centered within a maximum width of 1200px.
- Project grids will be responsively designed to display 3 columns on desktop, 2 on tablet, and a single column on mobile for optimal cross-device usability.
- Minimalist, line-art style icons will be utilized throughout the application, particularly for elements like the theme toggle and interactive controls, reinforcing the clean aesthetic.
- Animations will be subtle and functional, avoiding any flashy or distracting effects. Project cards will include a very soft scale-up effect (maximum 1.02) on hover for gentle user feedback, and transitions between pages or modal states will be understated.
